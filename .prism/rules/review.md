# PR Review Checklist

## Before Submitting
- [ ] `swift format lint --strict` passes
- [ ] `swift build --build-tests --explicit-target-dependency-import-check error` passes
- [ ] `swift test` — all green
- [ ] No `TODO` or `FIXME` left (unless tracked in `trackers/tech-debt.md`)
- [ ] No force unwraps (`!`) in library code
- [ ] No `@unchecked Sendable`
- [ ] PR title: conventional commit format

## Code Quality
- [ ] New public API has DocC documentation
- [ ] New public types are `Sendable`
- [ ] Error types conform to `LocalizedError`
- [ ] No hardcoded strings (use localization)
- [ ] Access control is minimal (internal by default)

## Architecture
- [ ] No circular module dependencies
- [ ] New dependency justified and approved
- [ ] Follows module patterns (see `architecture.md`)

## Testing
- [ ] New code has tests
- [ ] Tests follow naming convention: `test_method_condition_result`
- [ ] Error paths tested
- [ ] No flaky tests (no timing-dependent assertions)

## PrismUI Specific
- [ ] Uses design tokens, no hardcoded values
- [ ] Works across 4 themes
- [ ] Keyboard navigable
- [ ] VoiceOver accessible
- [ ] Preview provided

## PrismServer Specific
- [ ] Middleware is composable (no side effects in handler)
- [ ] Async I/O only, no blocking calls
- [ ] Input validated before processing
- [ ] Error responses use proper HTTP status codes

## Documentation
- [ ] DocC on new public APIs
- [ ] Mintlify page if new feature/concept
- [ ] CHANGELOG entry (or auto-generated by CI)
